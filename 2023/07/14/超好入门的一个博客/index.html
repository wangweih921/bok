快速入门一个博客(gitee+hexo)

是否为搭建一个博客而苦恼?

是否因为网上的教程多样化而烦躁?

是否看到教程中超多叫不上名字的名词而困扰?

是否被多种技术的复杂操作而被劝退?

是否看着别人超级漂亮的博客感觉比不上望而却步?

那么废话不多说,请跟着我的教程一步一步的用最容易实现的方式将你的博客搭建起来吧

1.环境

  1. 一个Gitee的账号

    注册就行

  2. Git

    安装就行

  3. nodejs

    安装+配环境

2. 目标

请大家设想下,你有一篇图文并茂的笔记,而且想让所有人都看到;

所以肯定不能放在自己电脑里,所以普遍做法是买个域名搞个服务器,东西放到自己服务器上;

但我们没有服务器,也没有这个时间金钱成本去搞这个,就想简简单单的放在个公共区;放在知识平台下固然可以,但没法装*了,而且平台存储起来没有结构,没有好看的样式,你这优美的排版自然就浪费了

所以我们选用代码托管平台中的pages功能,帮我们将源码”跑起来”

选用Gitee的原因也很简单,因为github连接起来太慢了,不是所有人上网都那么Scienc

还有一个问题就是实现图文并茂的代码可能实现起来没那么容易,不是前端出身或者就是简简单单想分享个技术,不过就是想好看点,没有必要花费时间去设计静态页面.

那么我们就要用到用nodejs写的hexo博客框架去快速生成,而hexo中有很多优美的主题,让我们的博客也能更好看

于是我们的大致方向就订好了,

写内容 –> hexo —> 好看的博客 —>git发送 —>gitee托管 —>pages展示

  7. hexo的目录结构

    以下是对目录结构中各文件和文件夹的简要说明：

    • _config.yml：Hexo的主配置文件，用于配置网站的基本信息、主题、插件等。

    • package.json：npm包管理文件，记录了Hexo所依赖的插件和工具。

    • scaffolds：模板文件夹，用于创建新文章、页面等的模板，可以自定义其中的模板文件。

    • source

      ：资源文件夹，包含所有的源码、Markdown文件、图片等。

      • _drafts：草稿文件夹，用于存放尚未发布的文章，这些文章不会被生成到最终的网站中。
      • _posts：文章文件夹，用于存放已发布的文章，这些文章会生成到最终的网站中。
      • _pages：页面文件夹，用于存放静态页面，如关于页面、联系页面等。
      • assets：资源文件夹，用于存放图片、CSS、JavaScript等其他资源文件。

    • themes：主题文件夹，存放不同主题的文件，每个主题都有自己的文件结构和样式。

    看着复杂?无所谓,了解下就行

  8. 那么现在可以开始写一篇博客了

  9. 当你写好你的内容后,此时还没将你的文章加载成网页花里胡哨的形式,此时仅仅是md文档,所有需要更新生成你的Hexo网站

4.gitee的部署

在将博客传到码云(gitee)之前,需要将gitee进行一个部署,以及对hexo进行部分配置,方便找到gitee并上传

使用的前提是已经安装好了git(去https://git-scm.com/downloads下一个安装包,一路next安装)

  1. 创建仓库以及配置

    按照步骤操作点击 + —>创建仓库,进入正常填写即可

  2. 配置git

    进入命令行窗口,将生成的指令复制进来执行(一行一行复制)

    推荐配置并使用密钥,实现免密码登录,可以看官方密钥生成教程

    • 生成公钥

    • 三次回车

    • 进入 C:\Users\主机名.ssh 目录，把 id_rsa.pub 里面的信息复制到码云的 SSH公钥 中即可

  3. 开启pages服务

    请注意顺序,一步一步来

    • 配置完成后进行,点击初始化readme,使仓库不再是空仓库

    • 点击管理–>仓库信息–>基本信息–>开源

    • 点击管理旁边的服务—>选择Gitee pages

      第一次可能需要实名审核,按照要求去做就好

    • 点击启动,会给出一个url,等我们将博客传到里面,直接访问这个地址就可以看到你的博客了

  4. 配置hexo

    • 进入博客文件夹执行指令

      Hexo 提供了快速方便的一键部署功能,上面是安装 hexo-deployer-git。

    • 打开博客文件夹,配置_config.yml文件,配置git的地址
